Lập Trình Hướng Đối Tượng trong JavaScript Link to heading

Lập trình hướng đối tượng (OOP - Object-Oriented Programming) là một phương pháp lập trình dựa trên các đối tượng, nơi mỗi đối tượng có các thuộc tính (properties) và phương thức (methods). JavaScript là một ngôn ngữ lập trình hỗ trợ lập trình hướng đối tượng, mặc dù cú pháp của nó khác biệt một chút so với các ngôn ngữ lập trình hướng đối tượng khác như Java hoặc C++.

Trong JavaScript, bạn có thể tạo đối tượng và sử dụng các tính năng OOP như kế thừa, đóng gói và đa hình. Dưới đây là các khái niệm cơ bản của OOP trong JavaScript.

1. Đối Tượng (Object) Link to heading

Đối tượng là một tập hợp các thuộc tính và phương thức. Mỗi đối tượng có thể chứa các giá trị (properties) và các hàm (methods) để thao tác với dữ liệu.

1.1. Tạo Đối Tượng Link to heading 

let person = {
  name: "Alice",
  age: 25,
  greet: function() {
    console.log("Hello, my name is " + this.name);
  }
};

person.greet();  // Output: Hello, my name is Alice

Trong ví dụ trên, person là một đối tượng với các thuộc tính name, age và một phương thức greet.

2. Constructor Function (Hàm Khởi Tạo) Link to heading

JavaScript cho phép bạn tạo đối tượng bằng cách sử dụng hàm khởi tạo (constructor function). Hàm khởi tạo là một hàm được sử dụng để tạo và khởi tạo các đối tượng.

2.1. Hàm Khởi Tạo Link to heading 

function Person(name, age) {
  this.name = name;
  this.age = age;
  this.greet = function() {
    console.log("Hello, my name is " + this.name);
  };
}

let person1 = new Person("Alice", 25);
let person2 = new Person("Bob", 30);

person1.greet();  // Output: Hello, my name is Alice
person2.greet();  // Output: Hello, my name is Bob

Hàm khởi tạo Person tạo ra các đối tượng mới với các thuộc tính name, age và phương thức greet.

2.2. Từ Khóa this Link to heading

Trong JavaScript, this đại diện cho đối tượng hiện tại mà phương thức đang được gọi.

3. Class trong JavaScript Link to heading

Từ phiên bản ECMAScript 6 (ES6), JavaScript đã hỗ trợ cú pháp class để dễ dàng làm việc với lập trình hướng đối tượng. class là một cách viết ngắn gọn và dễ hiểu hơn cho việc tạo hàm khởi tạo và các phương thức.

3.1. Định Nghĩa Class Link to heading 

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log("Hello, my name is " + this.name);
  }
}

let person1 = new Person("Alice", 25);
person1.greet();  // Output: Hello, my name is Alice

3.2. Các Phương Thức trong Class Link to heading

Các phương thức trong class giống như các phương thức trong đối tượng. Bạn có thể thêm bất kỳ phương thức nào vào bên trong class.

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log("Hello, my name is " + this.name);
  }

  sayAge() {
    console.log("I am " + this.age + " years old");
  }
}

let person2 = new Person("Bob", 30);
person2.greet();  // Output: Hello, my name is Bob
person2.sayAge(); // Output: I am 30 years old

4. Kế Thừa (Inheritance) Link to heading

Kế thừa cho phép một đối tượng “thừa hưởng” các thuộc tính và phương thức từ một đối tượng khác. Trong JavaScript, bạn có thể sử dụng từ khóa extends để kế thừa từ một class khác.

4.1. Kế Thừa với Class Link to heading 

class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(this.name + " makes a sound");
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name);  // Gọi hàm khởi tạo của lớp cha
    this.breed = breed;
  }

  speak() {
    console.log(this.name + " barks");
  }
}

let dog = new Dog("Buddy", "Golden Retriever");
dog.speak();  // Output: Buddy barks

Trong ví dụ trên, Dog kế thừa từ Animal và ghi đè phương thức speak của lớp cha.

4.2. Từ Khóa super Link to heading

Từ khóa super được sử dụng để gọi phương thức hoặc hàm khởi tạo của lớp cha.

5. Đóng Gói (Encapsulation) Link to heading

Đóng gói là một khái niệm trong OOP cho phép ẩn các chi tiết triển khai của đối tượng và chỉ hiển thị các phương thức công khai để tương tác với đối tượng. Trong JavaScript, bạn có thể sử dụng private và public để kiểm soát quyền truy cập.

5.1. Đóng Gói với Getter và Setter Link to heading 

class Person {
  constructor(name, age) {
    this._name = name;  // Biến private (quy ước dùng dấu _)
    this._age = age;
  }

  get name() {
    return this._name;
  }

  set name(value) {
    this._name = value;
  }

  get age() {
    return this._age;
  }

  set age(value) {
    if (value < 0) {
      console.log("Age cannot be negative");
    } else {
      this._age = value;
    }
  }
}

let person = new Person("Alice", 25);
console.log(person.name);  // Output: Alice
person.age = -5;  // Output: Age cannot be negative

6. Đa Hình (Polymorphism) Link to heading

Đa hình cho phép các đối tượng của các lớp khác nhau có thể trả về cùng một phương thức nhưng với hành vi khác nhau. Đa hình thường được thực hiện thông qua việc ghi đè phương thức (method overriding).

6.1. Đa Hình trong JavaScript Link to heading 

class Animal {
  speak() {
    console.log("Animal makes a sound");
  }
}

class Dog extends Animal {
  speak() {
    console.log("Dog barks");
  }
}

class Cat extends Animal {
  speak() {
    console.log("Cat meows");
  }
}

let dog = new Dog();
let cat = new Cat();

dog.speak();  // Output: Dog barks
cat.speak();  // Output: Cat meows

Trong ví dụ trên, phương thức speak được ghi đè trong các lớp con Dog và Cat, thể hiện tính đa hình.

7. Kết Luận Link to heading

Lập trình hướng đối tượng (OOP) trong JavaScript cung cấp một cách tiếp cận mạnh mẽ để tổ chức và quản lý mã nguồn của bạn thông qua các đối tượng. Các khái niệm cơ bản như đối tượng, lớp, kế thừa, đóng gói và đa hình giúp bạn xây dựng các ứng dụng JavaScript dễ duy trì và mở rộng.

Bằng cách sử dụng các cú pháp như class, constructor, extends, và super, bạn có thể triển khai OOP một cách dễ dàng trong JavaScript. Những kỹ thuật này sẽ giúp bạn viết mã rõ ràng, dễ bảo trì và dễ hiểu hơn. Quay lại Chuyển sang bài học tiếp theo
