Asynchronous JavaScript Link to heading

Trong JavaScript, xử lý bất đồng bộ (asynchronous) cho phép bạn thực thi các tác vụ mà không làm tắc nghẽn (block) các tác vụ khác. Điều này rất quan trọng khi làm việc với các tác vụ mất thời gian như tải dữ liệu từ máy chủ, đọc tệp, hoặc các tác vụ cần phải chờ đợi. JavaScript cung cấp nhiều cách để xử lý bất đồng bộ, bao gồm callbacks, promises, và async/await.

1. Lý Do Cần Asynchronous JavaScript Link to heading

JavaScript hoạt động theo mô hình đơn luồng, có nghĩa là chỉ có thể thực thi một tác vụ tại một thời điểm. Tuy nhiên, trong các ứng dụng web hiện đại, thường có những tác vụ mất thời gian, ví dụ như gọi API từ máy chủ, mà không thể làm gián đoạn luồng xử lý chính.

Asynchronous JavaScript giúp tránh tình trạng “block” khi thực thi các tác vụ này, cho phép các tác vụ khác tiếp tục được thực hiện trong khi chờ đợi.

2. Callbacks (Hàm Gọi Lại) Link to heading

Callback là một hàm được truyền vào một hàm khác như một đối số và được thực thi khi tác vụ bất đồng bộ hoàn thành. Tuy nhiên, việc sử dụng quá nhiều callbacks có thể dẫn đến “callback hell”, nơi mã trở nên khó đọc và khó bảo trì.

2.1. Ví Dụ về Callbacks Link to heading 

function fetchData(callback) {
  setTimeout(function() {
    let data = "Dữ liệu đã được tải";
    callback(data);
  }, 2000); // Giả lập việc tải dữ liệu mất 2 giây
}

fetchData(function(data) {
  console.log(data);  // Output: Dữ liệu đã được tải
});

Trong ví dụ trên, fetchData là một hàm bất đồng bộ và khi nó hoàn thành, hàm callback sẽ được gọi.

2.2. Callback Hell Link to heading

Khi bạn có nhiều callback lồng nhau, mã có thể trở nên khó đọc và bảo trì:

fetchData(function(data) {
  console.log(data);
  fetchData(function(data2) {
    console.log(data2);
    fetchData(function(data3) {
      console.log(data3);
    });
  });
});

Đây là “callback hell”, nơi bạn phải lồng nhiều hàm callback, làm cho mã trở nên phức tạp.

3. Promises (Lời Hứa) Link to heading

Promises cung cấp một cách tiếp cận tốt hơn để xử lý bất đồng bộ bằng cách cho phép bạn xử lý kết quả của các tác vụ bất đồng bộ mà không phải sử dụng các callback lồng nhau. Promise đại diện cho một tác vụ bất đồng bộ có thể hoàn thành hoặc thất bại.

3.1. Cấu Trúc của Promise Link to heading

Một Promise có thể ở một trong ba trạng thái:

  • Pending: Chưa hoàn thành.
  • Resolved (Fulfilled): Hoàn thành thành công.
  • Rejected: Hoàn thành thất bại.

3.2. Ví Dụ về Promise Link to heading 

let promise = new Promise(function(resolve, reject) {
  let success = true;
  if (success) {
    resolve("Tải dữ liệu thành công");
  } else {
    reject("Tải dữ liệu thất bại");
  }
});

promise.then(function(result) {
  console.log(result);  // Output: Tải dữ liệu thành công
}).catch(function(error) {
  console.log(error);   // Output: Tải dữ liệu thất bại
});

Trong ví dụ trên, Promise thực thi một tác vụ bất đồng bộ và sử dụng then để xử lý khi thành công và catch để xử lý khi thất bại.

3.3. Chaining Promises (Liên kết Promise) Link to heading

Bạn có thể liên kết các promise với nhau để xử lý các tác vụ bất đồng bộ theo chuỗi:

let promise = new Promise(function(resolve, reject) {
  resolve("Tải dữ liệu thành công");
});

promise.then(function(result) {
  console.log(result);  // Output: Tải dữ liệu thành công
  return "Tiến hành xử lý dữ liệu";
}).then(function(result) {
  console.log(result);  // Output: Tiến hành xử lý dữ liệu
}).catch(function(error) {
  console.log(error);
});

4. Async/Await Link to heading

async và await là cú pháp hiện đại được giới thiệu trong ECMAScript 2017 (ES8), giúp xử lý bất đồng bộ dễ dàng hơn và giúp mã của bạn dễ đọc hơn so với việc sử dụng promise hoặc callback.

  • async: Được sử dụng để khai báo một hàm bất đồng bộ.
  • await: Được sử dụng trong hàm async để chờ đợi một promise.

4.1. Ví Dụ về Async/Await Link to heading 

async function fetchData() {
  let promise = new Promise(function(resolve, reject) {
    setTimeout(() => resolve("Dữ liệu đã được tải"), 2000);
  });

  let result = await promise;  // Chờ cho promise hoàn thành
  console.log(result);  // Output: Dữ liệu đã được tải
}

fetchData();

Trong ví dụ trên, hàm fetchData được khai báo là async, và bên trong nó sử dụng await để đợi cho promise hoàn thành trước khi tiếp tục thực thi.

4.2. Xử Lý Lỗi với Async/Await Link to heading

Bạn có thể sử dụng try/catch để xử lý lỗi trong các hàm bất đồng bộ sử dụng async và await.

async function fetchData() {
  try {
    let promise = new Promise(function(resolve, reject) {
      setTimeout(() => reject("Có lỗi xảy ra"), 2000);
    });

    let result = await promise;  // Chờ cho promise hoàn thành
    console.log(result);
  } catch (error) {
    console.log(error);  // Output: Có lỗi xảy ra
  }
}

fetchData();

5. Các Phương Thức Hữu Ích Khi Làm Việc Với Asynchronous JavaScript Link to heading

5.1. Promise.all() Link to heading

Promise.all() giúp thực thi nhiều promise cùng một lúc và chỉ trả về kết quả khi tất cả các promise đều hoàn thành thành công.

let promise1 = Promise.resolve("Dữ liệu 1");
let promise2 = Promise.resolve("Dữ liệu 2");
let promise3 = Promise.resolve("Dữ liệu 3");

Promise.all([promise1, promise2, promise3]).then(function(values) {
  console.log(values);  // Output: ["Dữ liệu 1", "Dữ liệu 2", "Dữ liệu 3"]
});

5.2. Promise.race() Link to heading

Promise.race() giúp trả về kết quả của promise nào hoàn thành trước tiên.

let promise1 = new Promise(function(resolve, reject) {
  setTimeout(resolve, 2000, "Dữ liệu 1");
});
let promise2 = new Promise(function(resolve, reject) {
  setTimeout(resolve, 1000, "Dữ liệu 2");
});

Promise.race([promise1, promise2]).then(function(value) {
  console.log(value);  // Output: Dữ liệu 2
});

6. Kết Luận Link to heading

Asynchronous JavaScript là một phần quan trọng trong việc phát triển các ứng dụng web hiện đại. Việc sử dụng các phương thức như callbacks, promises, và async/await giúp bạn xử lý các tác vụ bất đồng bộ một cách hiệu quả và dễ dàng.

  • Callbacks: Được sử dụng để thực thi một hàm sau khi tác vụ bất đồng bộ hoàn thành.
  • Promises: Cung cấp một cách tiếp cận dễ đọc hơn cho việc xử lý bất đồng bộ và tránh callback hell.
  • Async/Await: Cú pháp hiện đại giúp mã bất đồng bộ trở nên dễ đọc và dễ hiểu hơn.

Bằng cách sử dụng các kỹ thuật này, bạn có thể viết mã JavaScript bất đồng bộ mà không gặp phải các vấn đề về hiệu suất hay bảo trì. Quay lại Chuyển sang bài học tiếp theo
