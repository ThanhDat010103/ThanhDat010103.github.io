Xử Lý Ngoại Lệ (Exception Handling) trong Java Link to heading

Trong Java, Ngoại Lệ (Exception) là những sự kiện không mong muốn xảy ra trong quá trình thực thi chương trình, có thể gây ra sự gián đoạn hoặc lỗi trong chương trình. Java cung cấp cơ chế Xử lý Ngoại Lệ (Exception Handling) để giúp chương trình tiếp tục chạy ngay cả khi gặp phải lỗi.

Các Kiểu Ngoại Lệ Link to heading

Java phân loại các ngoại lệ thành hai nhóm chính:

  1. Checked Exceptions (Ngoại Lệ Kiểm Tra):

    • Là những ngoại lệ mà trình biên dịch yêu cầu phải xử lý hoặc khai báo trong mã nguồn.
    • Ví dụ: IOException, SQLException.
    • Các ngoại lệ này cần được bao bọc trong khối try-catch hoặc khai báo với từ khóa throws.

  2. Unchecked Exceptions (Ngoại Lệ Không Kiểm Tra):

    • Là những ngoại lệ không cần khai báo hoặc xử lý ngay lập tức.
    • Thường là các lỗi lập trình như chia cho số 0, truy cập mảng ngoài giới hạn, v.v.
    • Ví dụ: NullPointerException, ArrayIndexOutOfBoundsException.

Cấu Trúc Xử Lý Ngoại Lệ Link to heading

Cấu trúc cơ bản của xử lý ngoại lệ trong Java sử dụng try, catch, finally như sau:

try {
    // Mã có thể gây ra ngoại lệ
} catch (ExceptionType e) {
    // Xử lý ngoại lệ
} finally {
    // Mã này luôn được thực thi (bất kể ngoại lệ có xảy ra hay không)
}
  • try block: Chứa mã có thể gây ra ngoại lệ.
  • catch block: Xử lý ngoại lệ nếu có.
  • finally block: Mã trong finally luôn được thực thi, dù có xảy ra ngoại lệ hay không (thường dùng để đóng tài nguyên, như file, kết nối cơ sở dữ liệu).

Ví Dụ về Xử Lý Ngoại Lệ Link to heading 

public class ExceptionHandlingExample {
    public static void main(String[] args) {
        try {
            // Thử chia cho 0 sẽ gây ra ArithmeticException
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            // Xử lý ngoại lệ
            System.out.println("Lỗi: Không thể chia cho 0!");
        } finally {
            // Khối finally luôn được thực thi
            System.out.println("Khối finally luôn được thực thi.");
        }
    }
}

Kết quả:

Lỗi: Không thể chia cho 0!
Khối finally luôn được thực thi.

Ném Ngoại Lệ (Throwing Exceptions) Link to heading

Java cho phép lập trình viên ném ngoại lệ bằng cách sử dụng từ khóa throw. Điều này giúp bạn có thể tạo và ném các ngoại lệ tùy chỉnh khi cần thiết.

public class ThrowExample {
    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (Exception e) {
            System.out.println(e.getMessage());
        }
    }

    public static void checkAge(int age) {
        if (age < 18) {
            throw new IllegalArgumentException("Tuổi phải lớn hơn hoặc bằng 18");
        }
    }
}

Kết quả:

Tuổi phải lớn hơn hoặc bằng 18

Khai Báo Ngoại Lệ (Declaring Exceptions) Link to heading

Khi một phương thức có thể ném ngoại lệ, bạn cần khai báo ngoại lệ đó bằng từ khóa throws. Điều này giúp các phương thức gọi biết rằng có khả năng xảy ra ngoại lệ.

public class ThrowsExample {
    public static void main(String[] args) {
        try {
            readFile("invalidfile.txt");
        } catch (IOException e) {
            System.out.println("Lỗi: Không thể đọc tệp.");
        }
    }

    public static void readFile(String filename) throws IOException {
        // Giả sử phương thức này có thể ném IOException
        throw new IOException("Tệp không tồn tại.");
    }
}

Kết quả:

Lỗi: Không thể đọc tệp.

Kết luận Link to heading

Xử lý ngoại lệ là một phần quan trọng trong việc lập trình Java, giúp chương trình của bạn trở nên ổn định và dễ dàng kiểm soát các lỗi có thể xảy ra trong quá trình thực thi. Java cung cấp các cơ chế linh hoạt như try-catch, throw, throws, và finally để giúp lập trình viên dễ dàng xử lý ngoại lệ và đảm bảo chương trình tiếp tục chạy mượt mà.
