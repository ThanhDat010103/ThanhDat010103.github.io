Luồng (Threads) và Đa Luồng (Multithreading) trong Java Link to heading

Trong lập trình Java, luồng (thread) là một đơn vị thực thi nhỏ nhất mà hệ thống có thể xử lý. Mỗi chương trình Java chạy ít nhất một luồng (luồng chính), nhưng có thể tạo thêm nhiều luồng để xử lý các tác vụ đồng thời. Đa luồng (multithreading) cho phép thực hiện nhiều tác vụ cùng lúc trong một chương trình, làm tăng hiệu suất và phản ứng của ứng dụng, đặc biệt là với các ứng dụng yêu cầu tính toán nặng hoặc xử lý I/O đồng thời.

Luồng (Thread) trong Java Link to heading

Một luồng là một đơn vị thực thi trong một chương trình, giúp chương trình thực thi các tác vụ đồng thời. Java cung cấp một số cách để tạo và quản lý các luồng.

Cách tạo luồng trong Java Link to heading

Java cung cấp hai cách chính để tạo luồng:

  1. Kế thừa lớp Thread:
    • Tạo một lớp con kế thừa từ lớp Thread và ghi đè phương thức run().
    • Khi bạn gọi phương thức start(), phương thức run() sẽ được thực thi trong một luồng mới.
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread đang chạy: " + Thread.currentThread().getId());
    }
}

public class ThreadExample {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        
        // Bắt đầu thực thi luồng
        t1.start();
        t2.start();
    }
}

Kết quả:

Thread đang chạy: 11
Thread đang chạy: 12
  1. Cài đặt giao diện Runnable:
    • Tạo một lớp cài đặt giao diện Runnable và ghi đè phương thức run().
    • Sau đó, tạo một đối tượng Thread và truyền đối tượng Runnable vào khi khởi tạo.
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable đang chạy: " + Thread.currentThread().getId());
    }
}

public class RunnableExample {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread t1 = new Thread(myRunnable);
        Thread t2 = new Thread(myRunnable);

        // Bắt đầu thực thi luồng
        t1.start();
        t2.start();
    }
}

Kết quả:

Runnable đang chạy: 13
Runnable đang chạy: 14

Đa Luồng (Multithreading) Link to heading

Đa luồng (multithreading) là khả năng chạy nhiều luồng đồng thời trong một ứng dụng, giúp cải thiện hiệu suất khi thực hiện các tác vụ không đồng bộ hoặc yêu cầu tính toán nặng.

Lợi ích của Đa Luồng Link to heading

  1. Hiệu quả trong việc xử lý đồng thời: Đặc biệt hữu ích với các tác vụ như đọc/ghi tệp, truy vấn cơ sở dữ liệu, và các tác vụ tính toán nặng.
  2. Cải thiện phản hồi của ứng dụng: Các luồng khác có thể tiếp tục thực hiện ngay cả khi một luồng đang chờ đợi một tác vụ I/O hoàn thành.
  3. Tăng tốc độ cho các hệ thống đa nhân: Với hệ thống nhiều nhân (multi-core), các luồng có thể chạy đồng thời trên các nhân khác nhau.

Quản lý luồng Link to heading

  • Thread.sleep(milliseconds): Tạm dừng luồng hiện tại trong một khoảng thời gian.
  • Thread.yield(): Đưa luồng hiện tại vào trạng thái chờ, cho phép luồng khác được thực thi nếu có.
  • Thread.join(): Dừng luồng hiện tại cho đến khi luồng khác hoàn thành.

Ví dụ về Thread.sleep():

public class SleepExample {
    public static void main(String[] args) {
        try {
            System.out.println("Luồng sẽ tạm dừng 2 giây...");
            Thread.sleep(2000); // Tạm dừng 2 giây
            System.out.println("Luồng đã tiếp tục.");
        } catch (InterruptedException e) {
            System.out.println("Luồng bị gián đoạn.");
        }
    }
}

Kết quả:

Luồng sẽ tạm dừng 2 giây...
Luồng đã tiếp tục.

Ví dụ về Thread.join() Link to heading 

class MyThread extends Thread {
    public void run() {
        System.out.println("Thread " + Thread.currentThread().getId() + " bắt đầu.");
        try {
            Thread.sleep(1000); // Tạm dừng 1 giây
        } catch (InterruptedException e) {
            System.out.println("Luồng bị gián đoạn.");
        }
        System.out.println("Thread " + Thread.currentThread().getId() + " hoàn thành.");
    }
}

public class ThreadJoinExample {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        
        t1.start();
        try {
            t1.join(); // Chờ t1 hoàn thành trước khi tiếp tục
        } catch (InterruptedException e) {
            System.out.println("T1 bị gián đoạn.");
        }
        
        t2.start();
    }
}

Kết quả:

Thread 11 bắt đầu.
Thread 11 hoàn thành.
Thread 12 bắt đầu.
Thread 12 hoàn thành.

Tính Đồng Bộ trong Đa Luồng Link to heading

Khi nhiều luồng cùng truy cập và sửa đổi dữ liệu chung, có thể xảy ra các vấn đề như race condition hoặc data inconsistency. Để xử lý vấn đề này, Java cung cấp cơ chế đồng bộ hóa (synchronization) thông qua từ khóa synchronized.

Sử dụng synchronized để đồng bộ hóa Link to heading 

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

public class SynchronizationExample {
    public static void main(String[] args) {
        Counter counter = new Counter();

        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });

        t1.start();
        t2.start();

        try {
            t1.join();
            t2.join();
        } catch (InterruptedException e) {
            System.out.println("Lỗi gián đoạn.");
        }

        System.out.println("Giá trị cuối cùng của counter: " + counter.getCount());
    }
}

Kết quả:

Giá trị cuối cùng của counter: 2000

Kết luận Link to heading

Đa luồng (multithreading) là một kỹ thuật mạnh mẽ trong lập trình Java, giúp tối ưu hóa hiệu suất và tính phản hồi của ứng dụng. Java cung cấp nhiều công cụ và kỹ thuật để làm việc với luồng, bao gồm việc tạo luồng, đồng bộ hóa, và quản lý luồng hiệu quả. Khi làm việc với đa luồng, bạn cần chú ý đến các vấn đề như đồng bộ hóa và xử lý các ngoại lệ để đảm bảo ứng dụng của bạn hoạt động ổn định và chính xác. Quay lại Chuyển sang bài học tiếp theo
